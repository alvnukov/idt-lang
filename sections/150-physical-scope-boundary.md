## 150. Physical Scope Boundary

Status:

`physical_scope_boundary_declared`

This section fixes the public positioning of the programme.

The current object is not a completed derivation of physical constants from
inherited distinguishability alone. It is a reconstruction framework with
explicit anchor requirements, no-refit gates, and dimensionless tests.

### 150.1. Decisive diagnostic

The current primitive layer has no accepted primitive source of mass dimension.

The only manifest symbols carrying a nonzero \(M\) exponent are not primitive
physical anchors:

1. `A0_I = blocked`;
2. `hbar_I = blocked`;
3. `primitive_mass_anchor_I = open`;
4. `primitive_work_unit_I = open`;
5. `kappa_chi_I = open`;
6. `G_I = target`;
7. `G_N`, `hbar_obs` are experimental gates;
8. energy/momentum operators are only conditional readouts and depend on the
   blocked action scale.

Therefore the current primitive set cannot derive:

\[
M,
\qquad
ML^2T^{-1},
\qquad
ML^2T^{-2},
\qquad
M^{-1}L^3T^{-2}.
\]

This is not a missing calculation. It is a dimensional obstruction.

### 150.2. Current route limits

The update-spectrum route can at most improve the dimensionless phase-step
structure:

\[
\Theta_*=\zeta_{\mathrm{step}}.
\]

It does not lock an absolute tick scale.

The inertial-response route gives:

\[
m=\frac{J}{\Delta v}.
\]

It does not lock mass unless \(J\) is independently fixed.

The spectral-kernel phase-cost route gives a possible dimensionless no-refit
test:

\[
\lambda_{\theta C}
=
\frac{\Theta_\gamma}{\bar C_\gamma^K}.
\]

It does not derive \(A_{0,I}\), \(W_{0,I}\), \(\tau_{0,I}\), or \(\hbar_I\).

### 150.3. Public positioning

The public claim is not:

`completed_physical_theory_from_current_primitives`.

The public claim is:

`formal_reconstruction_framework_with_anchor_protocols`.

The following stronger claims remain out of scope until an independent
dimensionful anchor is supplied:

1. `hbar_I = derived`;
2. `G_I = derived`;
3. `dark_sector_or_galaxy_explanation_I = derived`;
4. `completed_QM_GR_replacement_I = derived`.

The defensible object is:

`formal_reconstruction_language = still_viable`.

That object may organize readout structures, anti-fit gates, finite verifier
checks, and dimensionless no-refit tests. It must not be presented as a theory
that derives physical constants.

### 150.4. What implementation could unlock

The positive target is a staged physical reconstruction programme.

A successful implementation would not merely add notation. It would provide:

1. a typed boundary between primitive structure and physical readout;
2. finite verifier checks for the internal consistency of candidate bridges;
3. pre-registered no-refit tests for dimensionless phase, cost, and residual
   relations;
4. calibrated quantum and weak-gravity branches whose anchors are declared
   once and then held fixed;
5. a way to compare failed bridges, near misses, and surviving candidates under
   the same manifest discipline;
6. a compact research artifact that collaborators can audit, extend, or
   falsify;
7. a claim-bounded interpretive language that can be adapted to AI scientific
   reasoning agents.

The main advantage is therefore not a premature claim of completion. The main
advantage is a controlled path from foundational primitives to executable
constraints, where every bridge must say whether it is derived, calibrated,
conditional, or still open.

For AI systems, that path is valuable because it can turn physical theory into
a navigable representation of the world. Instead of treating theory text as
unstructured prose, an agent could use the IDT manifest to track which
observations are readouts, which structures are primitive, which constants are
anchors, which bridges are testable, and where new evidence would change the
map.

### 150.5. Anchor policy

Any later dimensionful physical branch must declare one independent anchor
source, for example:

1. a primitive mass/impulse/work anchor;
2. a primitive action quantum;
3. an independently derived dimensional source outside the current primitive
   list.

If such an anchor is calibrated rather than derived, the branch must say so
explicitly and treat the calibrated quantity as an assumption, not as a result.
This keeps the reconstruction programme publishable without overstating it.

### 150.6. Role taxonomy policy

Before a future physical bridge is evaluated, its main objects must be assigned
sector roles:

1. structural selector;
2. dimensional anchor;
3. dimensionless coupling;
4. bridge assumption;
5. derived readout;
6. experimental gate;
7. blocked claim.

This is now a required precondition for extending the programme to EM, weak,
strong, or gravity-sector claims.

### 150.7. Stop rule

No future section may claim progress toward `hbar_I = derived`, `A0_I =
derived`, or `G_I = derived` unless it first introduces and justifies an
independent \(M\)-dimension source or explicitly narrows the theory to a
dimensionless reconstruction programme.

If neither happens, further algebraic expansion is classified as:

`unanchored_dimensionful_elaboration`.
